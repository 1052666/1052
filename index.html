03 / FLOW / 一次完整的执行链路

从一句话，到真的完成任务。 From one sentence to actual execution.

STEP 01

接收入口

Input arrives

请求可以来自网页聊天、任务回调、扫码微信、桌面微信群聊、飞书等入口，统一进入同一套 Agent 运行时。

STEP 02

拼接上下文

Context assembled

系统提示词、权限模式、群提示词、群记忆、长期记忆、输出配方和最近历史会按场景动态注入。

STEP 03

挂载工具

Tools mounted

P0 先决定是否需要搜索、文件、记忆、Wiki、计划或渠道能力，再按需挂载对应能力包，不把所有说明一股脑塞给模型。

STEP 04

执行与写回

Execution + writeback

工具事件会实时回传到前端；任务完成后再把结果写回聊天、通知中心、记忆或外部社交通道。

06 / FAQ / 常见问题

你大概率会先问这些。 The first questions most people ask.

Q 01 1052 OS 和普通 AI 聊天工具最大的区别是什么？+
普通聊天工具更像一个回答框；1052 OS 是一个本地 Agent 工作台。它能接入文件、任务、记忆、Wiki、搜索、终端和社交通道，并把执行状态持续写回系统。 It is a local agent workspace, not just a response box.
Q 02 默认模型配置是什么？+
当前设置页默认预设是 `1052 API`，Base URL 为 `https://api.lxj.asia/v1`，模型为 `deepseek-v4-flash-search`。API Key 获取地址是 `https://api.lxj.asia/register?aff=UOBG`。 The default profile is 1052 API with DeepSeek V4 Flash Search.
Q 03 微信桌面通道和原扫码微信通道有什么区别？+
桌面微信通道是独立的新社交通道板块，针对 Windows 桌面微信窗口自动化，支持群聊监听、群提示词、群记忆和主动发送；它不会和原扫码微信板块混在一起。 Desktop WeChat is a separate Windows-only channel, distinct from the older scan-login WeChat integration.
Q 04 数据默认存在哪里？+
运行时数据默认落在项目根目录下的 `data/`，包括聊天历史、记忆、Wiki、Notes、Resources、生成图片和渠道状态等。项目本身遵循本地优先。 Runtime data is stored locally in the project's data directory.
Q 05 工具调用卡住时现在怎么处理？+
新版本已经补上工具执行事件和单工具超时保护。前端会显示工具开始、完成或失败状态，工具不会再无限静默占住整条消息链路。 Tool calls now emit progress events and have timeout protection.
Q 06 能自定义扩展吗？+
可以。项目支持本地 Skills、UAPIs、文件系统工具、搜索和终端协作。你既可以挂新的能力包，也可以把它当作个人本地代理工作台继续深改。 Yes. The project is designed to be extended with skills, tools, and local integrations.
